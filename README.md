## Objective
Conduct a security monitoring data analysis of a financial institution’s log files to identify suspicious activity, detect potential threats, and evaluate monitoring effectiveness.

## Tools Used
- PowerShell  
- Log Analysis  
- HTTP/Web Server Logs  

## Key Activities
- Analyzed Access-1, Apache, and Access-2 log files to evaluate network behavior  
- Identified abnormal HTTP status codes (400, 404) and failed requests  
- Detected tunneling attempts using CONNECT requests  
- Investigated potential Shellshock exploit activity and bot traffic  
- Used PowerShell commands (Select-String, Measure-Object, Group-Object) to filter and analyze log data  

## Key Findings
- Repeated HTTP 400 errors indicating possible probing or malformed requests  
- High traffic from specific IP addresses suggesting automated bot activity  
- Evidence of potential Shellshock exploit attempts targeting system vulnerabilities
